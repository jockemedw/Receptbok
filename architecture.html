HTTP-anrop / databas-operation
Direkt från browser (JS SDK, ingen server-roundtrip)

1 — Öppna appen

Användaren besöker receptbok-six.vercel.app. Vercel serverar HTML och JS-moduler. Appen hämtar sedan recept och veckoplan direkt från Supabase via JS SDK — ingen server-roundtrip behövs för läsning.

2 — Generera matsedel

"Generera"-knappen skickar en POST till /api/generate. Servern läser recepthistorik och recept från Supabase, väljer dagarna deterministiskt (ingen AI), sparar sedan veckoplan och inköpslista tillbaka i Supabase.

3 — Prisoptimering

Om prisoptimering är påslagen hämtar /api/generate Willys veckans erbjudanden via ett öppet API (ingen inloggning krävs). Recept med extrapris-ingredienser prioriteras och visas med en grön sparbadge.

4 — Dispatch till Willys

Inköpslistan kan skickas direkt till willys.se/cart via /api/dispatch-to-willys. Kräver sessionscookies + CSRF-token, som fångas automatiskt av en Chrome-extension vid vanliga Willys-besök.

5 — Importera recept

Foto eller URL skickas till /api/import-recipe. Google Gemini (gratisnivå) tolkar innehållet och returnerar ett strukturerat recept som sparas i Supabase. Enda AI:n i systemet — receptval är alltid deterministiskt.

6 — Deploy

Git push till main triggar automatisk Vercel-deploy på ~30 sekunder. Supabase-databasen är helt fristående från koden — data påverkas aldrig av deployments. Inga manuella steg.

## 4. Implementationsförslag

### Datastruktur

Lägg till ett `seasons`-fält i `recipes.json` per recept:

```json
{
  "id": 7,
  "title": "Grönkålssallad med rostade rödbetor",
  "seasons": ["höst", "vinter"],
  "tags": ["vardag30", "veg", "sallad"],
  ...
}
```

Recept utan `seasons`-fält (eller med `seasons: []`) behandlas som neutrala.

### Integration i `selectRecipes()`

Ny parameter i constraints:

```javascript
// api/generate.js — inuti selectRecipes()
const { season_weight = false } = constraints;

// Bestäm aktuell säsong från startDate
function getCurrentSeason(dateStr) {
  const month = new Date(dateStr).getMonth() + 1; // 1-12
  if (month >= 3 && month <= 5) return "vår";
  if (month >= 6 && month <= 8) return "sommar";
  if (month >= 9 && month <= 11) return "höst";
  return "vinter";
}

// I pick()-funktionen, efter filtrering men före slumpmässigt val:
function weightedPick(pool, currentSeason, seasonWeight) {
  if (!seasonWeight) {
    // Ingen säsongsviktning — slumpmässigt som idag
    return pool[Math.floor(Math.random() * pool.length)];
  }

  // Bygg viktat urval
  const weighted = pool.map(recipe => {
    const seasons = recipe.seasons || [];
    if (seasons.length === 0) return { recipe, weight: 1 };
    if (seasons.includes(currentSeason)) return { recipe, weight: 2 };
    return { recipe, weight: 0.5 }; // Ur säsong = halverad chans
  });

  const totalWeight = weighted.reduce((sum, w) => sum + w.weight, 0);
  let rand = Math.random() * totalWeight;
  for (const { recipe, weight } of weighted) {
    rand -= weight;
    if (rand <= 0) return recipe;
  }
  return pool[pool.length - 1]; // Fallback
}
```

### Vikter

| Receptstatus | Vikt |
|-------------|------|
| Matchar aktuell säsong | 2.0x |
| Ingen säsongstagning (neutral) | 1.0x |
| Ur säsong (har tags men inte aktuell) | 0.5x |

### UI (opt-in toggle)

```html
<!-- I inställningspanelen -->
<label class="toggle-row">
  <span>Säsongsanpassning</span>
  <input type="checkbox" id="seasonWeight" />
</label>
```

### Beteende

- **Ingen hård filtrering** — säsongsrecept får dubbel vikt, men ingenting blockeras
- **Neutral = 1x** — recept utan säsongsdata påverkas inte
- **Ur säsong = 0.5x** — halverad chans, men kan fortfarande väljas
- **Sparar inte som preferens** — skickas per anrop (default av)
- **Interagerar med prisoptimering** — om ett rearecept ÄR i säsong, multiplikativt: 2x × rea-boost
- **Gränsfall**: Recept taggade "åretrunt" (alla 4 säsonger) → behandlas som neutral (1x)

### Migrering

Kör `scripts/season-analysis.mjs` och använd output för att lägga till `seasons`-fält.
Alternativt: lagra säsongstabellen i koden och beräkna dynamiskt vid varje anrop
(fördel: inga nya fält i recipes.json, nackdel: ~50ms extra per anrop).

**Rekommendation:** Statisk taggning i `recipes.json` — enkel, deterministisk, gratis.

### Hantering av ojämn fördelning

Fördelningen är naturligt skev: sommar och höst har flest recept (184 resp. 223), medan vår (45) och vinter (32) har färre. Detta speglar verkligheten — svenska råvaror har sin topp jun–nov.

**Praktisk konsekvens:** I vinter/vår-läge finns färre recept med 2x-vikt, men systemet fungerar ändå:
- De 32 vinter-recepten får 2x → prioriteras naturligt
- De 22 neutrala recepten (inga säsongsingr.) behåller 1x → fyller ut
- Sommar/höst-recepten får 0.5x → kan fortfarande väljas om historikfiltrering gör att pool:en krymper

Om poolen efter alla filter (historik, protein, veg, etc.) har <5 recept med vintertagg → `weightedPick` kan fortfarande välja bland alla (0.5x är inte 0x).

### Metodologi

1. **Ingrediensklassificering:** Varje ingrediensrad parsas → mängd/enhet strippas → namn matchas mot säsongstabell
2. **Pantry-exkludering:** ~80 skafferivaror (olja, kryddor, pasta, ris, socker, etc.) räknas aldrig
3. **Fryst/konserverat:** Ingredienser med "frysta", "tinade", "burk", "torkade" exkluderas (tillgängliga året runt)
4. **Örtviktning:** Krukörter (persilja, koriander, basilika, dill, mynta) ges 0.3x vikt — de säljs färska året runt i svenska butiker
5. **Säsongspoäng:** Andel av viktade säsongsingredienser som är i säsong per kvartal. Tröskel ≥60% för taggning.
6. **Neutrala recept:** Recept utan matchade säsongsingredienser får 0.5 (baseline) → ingen tagg → 1x i viktningen

### Kända begränsningar

- Tomater: "6 tomater" räknas som sommar (Jul-Sep, svenskt friland), men i praktiken köps importerade tomater året runt. Detsamma gäller gurka, paprika. Valet att räkna dem som säsongsvaror speglar *kvalitet* snarare än *tillgänglighet*.
- Svamp: "blandad svamp" matchas mot vild svamp (Aug-Okt), men butikssvamp (champinjoner, shiitake) odlas året runt. Recept som specificerar "champinjoner" eller "shiitake" får korrekt åretrunt-behandling.
- Proteinbalans: Kyckling, nötkött, fläsk, lax räknas som åretrunt. Torsk (Jan-Apr) och lamm (Maj-Okt) har säsongsvariation som påverkar poängen.
